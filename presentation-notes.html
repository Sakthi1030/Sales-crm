Nexus CRM Presentation Notes

Brief explanation of the project, file structure, data source, and how to present it clearly.

1. Project Overview

Nexus CRM is a full-stack CRM application built using React, Node.js, Express, and PostgreSQL. It helps manage sales work such as leads, customers, deals, tasks, reports, and team access.

2. Main Project Parts

Frontend

frontend/ contains the user interface shown in the browser.

Backend

backend/ contains the API, route handling, authentication, and business logic.

Database

database/ contains the SQL schema and seed script for sample data.

3. Important Files and Their Functions

  • backend/server.js starts the Express server and loads all API routes.
  • backend/config/db.js connects the backend to PostgreSQL.
  • backend/middleware/auth.js handles JWT auth and admin protection.
  • backend/routes/*.js handles features like auth, leads, customers, deals, tasks, reports, and team.
  • database/schema.sql defines tables, keys, indexes, and constraints.
  • database/seed.js inserts sample demo records into the database.
  • frontend/src/App.jsx defines app routing, protected screens, and layout.
  • frontend/src/context/AuthContext.jsx manages login state and token handling.
  • frontend/src/utils/api.js keeps frontend API calls in one place.
  • frontend/src/pages/*.jsx contains screens like Dashboard, Leads, Customers, Pipeline, Tasks, Reports, and Team.

4. Where the Data Comes From

The current project data comes from database/seed.js. That script generates demo users, leads, customers, deals, interactions, and tasks.

The data is not from a real company and not from an external live API. It is seeded sample data stored in PostgreSQL.

5. Is the Data Real-Time or Static?

The best answer is: it is dynamic and database-driven, but not external real-time data.

  • It is not just hardcoded screen data.
  • It is stored in PostgreSQL, so changes are saved.
  • It updates when the user creates or edits records.
  • It is not live streaming data from the outside world.

6. Authentication and Roles

The app uses JWT authentication with two main roles:

  • Admin with full access
  • Sales Rep with more limited access

Role handling exists in both frontend routing and backend middleware.

7. Simple Architecture Flow

Browser UI → Express API → PostgreSQL Database → Express API → Browser UI

The frontend never talks directly to PostgreSQL. All requests go through the backend.

8. Best Presentation Summary

Nexus CRM is a full-stack CRM system built with React, Node.js, Express, and PostgreSQL. The frontend handles the interface, the backend handles API logic and authentication, and PostgreSQL stores the CRM data. The project uses seeded demo data, so it is interactive and persistent, but not external real-time data.
